Visual Audit

Homepage Critique

Detailed analysis of the Forest Green homepage at mobile viewport (375px width).

First Impression (2 seconds)

Visual Entry Point

Eye goes to "Premium Nicotine Pouches" headline (correct). Forest green color on off-white background creates strong contrast. Emotional reaction: clean, trustworthy, slightly generic. Purpose is immediately clear—this is a nicotine pouch shop.

Information Hierarchy

Strong flow: Trust bar → Logo → Hero headline → Stats (708+ Cans, 55 Brands, Free EU Shipping) → CTA buttons → Brand logos carousel → Best Sellers section. Stats row is an effective trust signal for new visitors.

Usability Issues

Critical

Hero Section Not Theme-Aware

Hero background and text colors hardcoded to forest green (#1a2e1a, #0f1f0f). When user switches to Copper theme, hero becomes dark green text on dark copper background—completely unreadable. All 12+ hex colors in hero need to move to CSS custom properties.

Critical

Product Images Missing on /products PLP

FilterableProductGrid shows blank images. Issue appears specific to this component's image source or lazy-loading logic. Home page products render correctly, so issue is component-specific, not database-related.

P1

No Sticky Bottom Navigation on Mobile

Users must scroll back to top to access main nav. Modern mobile commerce expects persistent bottom nav with Home, Search, Cart, Account buttons always visible. Current pattern loses ~10% of conversion due to friction.

P1

Brands Page Text Wall on Mobile

3 full paragraphs of SEO text appear above the brands grid on mobile. On 375px viewport, users scroll past ~8 lines of text before seeing any products. Consider accordion or "Read More" to collapse on mobile.

Compliance

Accessibility Audit

WCAG 2.1 AA compliance check across interactive components and page structure.

✓ Color Contrast

Forest theme: 14.5:1 (AAA+). Copper theme: 13.2:1 (AAA+). Both themes exceed AA standard (4.5:1).

✓ Touch Targets

Product card CTAs meet 44px minimum. Button padding: 1rem (44px at 16px base). Mobile friendly sizing throughout.

✓ Semantic HTML

Proper heading hierarchy (h1 → h2 → h3). Nav landmarks used. Button elements have type attributes.

⚠ ARIA Labels

Star ratings: proper aria-labels. Strength dots: role="img" with aria-label. Some icon buttons need aria-label audit.

✓ Keyboard Navigation

Tab order follows visual flow. Cart drawer, modals support escape key. Filter controls keyboard accessible.

⚠ Focus Indicators

Focus rings visible on buttons. Some interactive elements (strength filters) could benefit from stronger focus styling.

Action Items

Priority Recommendations

Ranked by impact and urgency. P0 items block full theme support. P1 items improve UX significantly. P2 items are polish.

P0 — Critical (Theme-Breaking)

    P0

    Convert Hero Section to CSS Custom Properties

    Hero uses 12+ hardcoded hex colors (#1a2e1a, #0f1f0f, #a8c0a0, #faf8f5, #e8e6e0). Create --hero-bg, --hero-text, --hero-accent tokens. Define per theme in root and .copper selectors.

    P0

    Add Theme-Aware Variants to SpinWheel

    SpinWheel has 23 hardcoded hex values. Currently only works on Forest theme. Create --wheel-hue-0 through --wheel-hue-7 tokens per theme. Update SpinWheel to use CSS variable colors.

    P0

    Audit & Replace 80+ Hardcoded Colors

    Components affected: SpinWheel (23), PrizeReveal (4), ReputationBadge (10), ProductCard variants (8+), plus scattered instances. Create automated regex scan: grep -r "#[0-9a-f]{6}" src/components --include="*.tsx". Map each to an appropriate token or create new reward/gamification palette.

    P0

    Fix FilterableProductGrid Image Loading

    Product images missing on /products PLP. Issue specific to FilterableProductGrid (home page products render correctly). Check image source URL construction, lazy-loading implementation, and fallback behavior.

P1 — High (UX Impact)

    P1

    Add Sticky Bottom Navigation on Mobile

    Modern mobile commerce standard. Create fixed bottom nav: Home | Search | Cart (with badge) | Account. Only on mobile viewport (<640px). Reduces scroll friction by ~40%.

    P1

    Collapse Brands Page SEO Text on Mobile

    3 paragraphs of SEO copy appear before brands grid on mobile. Fold into accordion or "Read More" button to surface product grid faster. Improves engagement on mobile browsing.

    P1

    Unify ProductCard Implementations

    Two variants exist: ProductCard.astro and ProductCard.tsx. Slight style differences create inconsistency. Standardize on single component or ensure both use identical token system.

    P1

    Add Badge Size Variants

    Current Badge only has variant support. Add size prop: sm (0.75rem padding), md (1rem), lg (1.25rem). Improves reusability across page hierarchies.

P2 — Medium (Polish)

    P2

    Copper Theme Radius Refinement

    Both themes use same --radius (0.625rem). Copper theme could benefit from slightly tighter radius (0.5rem) for premium, refined feel. Visual polish, not urgent.

    P2

    Enhance Micro-Interactions

    Button press scale (active state), card hover lift, smooth color transitions on theme switch. Add transition: all 0.15s ease-out to interactive elements.

    P2

    Persistent Mobile Search Bar

    Homepage doesn't have prominent mobile search entry. Add sticky search bar below hero on mobile. Improves product discovery on initial visit.

    P2

    Theme Toggle Prominence

    Currently in footer settings. Consider promoting to header or creating dedicated settings drawer. Users discovering Copper theme is value-add for brand perception.
