Impact

On Copper theme, the hero stays Forest green. The dark green hero on dark copper background creates zero visual contrast. The hero blends into the rest of the page, destroying the hero section's visual hierarchy.

Fix

Replace all 12+ hardcoded hex values with CSS custom properties:

Add to index.css for both .forest and .copper: --hero-bg: hsl(...) --hero-gradient-from: hsl(...) --hero-gradient-via: hsl(...) --hero-text-primary: hsl(...) --hero-text-muted: hsl(...) --hero-cta-bg: hsl(...) --hero-cta-text: hsl(...)

Summary

Total Findings: 14 (3 critical, 5 high, 3 medium, 2 low, 1 observation)

The core issue is 80+ hardcoded hex values throughout the codebase, nearly all from the Forest theme. When Copper theme is enabled, these hardcoded values do not adapt, creating a jarring visual disconnect especially on critical UI elements (hero sections, spin wheel, brand headers).

Root Cause: The design system tokens (--primary, --primary-foreground, --color-tier-*, --color-rarity-*) are correctly defined in index.css for both themes, but many components were built with hardcoded colors during development and never refactored to use tokens.

Path Forward: Systematic refactoring to replace hardcoded colors with theme tokens. The existing token infrastructure is solid; this is a codebase cleanup task, not a design problem. Priority order prioritizes visual impact: heros and spin wheel first (P0), then brand headers and blog (P1), then smaller components (P2/P3).
