✅ Confirmed: Astro is the right choice

SEO Win

Info/content pages ship 0KB JavaScript. Product catalog pages render as static HTML. Google crawls everything instantly. This is existential for your nicotine pouch niche (no paid ads possible).

White-Label Ready

Config-driven tenant system with CSS custom properties. One codebase → multiple branded storefronts. Turborepo monorepo when tenant #2 arrives. Vercel Platforms handles multi-domain.

shadcn/ui Works

Official Astro support. All 52 shadcn components carry over unchanged. Simple components use client:load, portal-based ones use client:only="react".

Performance

Lighthouse 95-100 for static pages. 50-80% JS reduction vs current SPA. LCP improvements of 2-5 seconds on mobile. React only loads where islands exist.

⚠️ Key Challenges Identified

1. React Context doesn't cross islands

Cart, Theme, Wishlist, Language, CookieConsent — all 6 contexts must become Nanostores (atomic stores shared across islands). This is mechanical but touches every component that reads context.

2. Radix portal components need client:only

~30 of 52 shadcn components (Dialog, Sheet, Select, Popover, etc.) cause hydration mismatches in SSG. Must use client:only="react" which means no server-rendered HTML for those islands. Fine because these are interactive widgets, not SEO content.

3. Sonner toasts across islands

Single <Toaster> in layout + CustomEvent bridge or Nanostores for cross-island toast triggering. Not hard, just different from the current global provider pattern.

4. No client-side routing by default

Full page navigations instead of React Router. Astro View Transitions API smooths this out with page morphing and persistent elements, but it's different from SPA navigation.

🏗️ Architecture Decisions (Locked In)

Approach: Full Astro — rewrite pages as .astro, React only for interactive islands State: Nanostores — framework-agnostic, ~1KB, purpose-built for islands Data: Hybrid SSG+SSR — static HTML for catalog, client overlay for live pricing Components: shadcn/ui stays — zero source changes, just wrapped in client directives Styling: Tailwind + CSS custom properties — unchanged, framework-agnostic Backend: Supabase edge functions — zero changes needed Deploy: Vercel + @astrojs/vercel adapter — GitHub Actions CI/CD White-label: Tenant config object → CSS vars + feature flags → Turborepo later
